# AgileClaw Migration Plan (2026-03-02)

## 목표

OpenClaw 수준의 플랫폼 범위를 추구하지 않고, NanoClaw 철학 기반의 최소 코어 + 확장형 구조로 전환한다.

## 범위

포함:
- 코어 아키텍처 재정의
- 도구 플러그인 자동 로딩
- 스킬 카탈로그 반영
- 액션 기반 크론 라우팅

제외:
- 멀티채널 대규모 게이트웨이
- 별도 웹 대시보드
- 멀티에이전트 분산 실행

## 단계별 계획

## Phase 0: 아키텍처 고정 (완료 목표: 즉시)

- 문서 4종 재작성
- 목표/비목표 명시
- 수용 기준 정의

## Phase 1: 코어 리팩터링 (완료 목표: 이번 사이클)

- `tools/*.py` 자동 로딩
- `skills/*/SKILL.md` 카탈로그
- 크론 액션 라우팅

## Phase 2: KPI 자동 측정 (완료)

- Threads 도구
- Reddit 도구
- 일일 점검 스킬

## Phase 3: 자동 리포트 운영화 (진행 중)

- 주간 리포트 포맷/송신
- 실패 재시도/알람 정책

## 리스크 및 대응

1. 리스크: 문서-코드 불일치 재발
- 대응: 기능 변경 시 동일 턴에 문서 동시 수정

2. 리스크: 확장 포인트 난립
- 대응: 도구 규약(`get_tool_specs`) 표준화

3. 리스크: KPI 측정 API 변경
- 대응: 도구 단위 분리 + 개별 테스트

## 성공 기준

- 코어는 단순하고 읽기 쉬워야 한다.
- 새 기능은 코어 수정 최소화로 추가되어야 한다.
- 에자일 루프가 매주 자동 실행되어 KPI 갭을 보고해야 한다.
